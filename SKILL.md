## Critical Rules (Never Forget)

- **`'use client'` boundary** ŌĆö Only add when you need browser APIs, hooks, or interactivity. Keep everything else server-side.
- **Server Components can't use hooks** ŌĆö No `useState`, `useEffect`, `useContext` in server components.
- **Colocate data fetching** ŌĆö Fetch data in the component that needs it, not prop-drill from the top.
- **Zod for all schema validation** ŌĆö Never trust raw form/API data without validation.
- **Tailwind = utility-first** ŌĆö Don't fight it. Compose utilities, don't fight the cascade.
- **Server Actions = mutations** ŌĆö Use them for form submits and data mutations; fetch for reads.
- **`<Image>` over `<img>`** ŌĆö Always use Next's Image for automatic optimization and CLS prevention.
- **Abort in-flight requests** ŌĆö Use `AbortController` to prevent race conditions on navigation.
- **Types in, types out** ŌĆö Mirror backend response types exactly. No `any` from API calls.

## Pro Tips

- Use `create-next-app --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"` for a fully typed Next.js project
- Run `npx tsc --noEmit` to type-check without building
- Use `shadcn@latest add <component>` to add components ŌĆö they live in your codebase, not node_modules
- Use `next.config.ts` for all Next.js configuration (not `.js`)
- Use `react-hook-form` + `zod` for all forms ŌĆö never manage form state with useState
- Enable React Compiler in `next.config.ts` with `reactCompiler: true` ŌĆö eliminates most `useMemo`/`useCallback` manually
- Use Biome (`npx biome check`) for linting ŌĆö 10ŌĆō100x faster than ESLint
- Run `npx biome migrate --write` after every Biome upgrade to migrate your config for breaking changes
- Use `use cache` for all server-side data fetching in Next.js 16 ŌĆö it's the explicit opt-in caching model

Start with `setup.md` to initialize a project, then `components.md` for shadcn/ui patterns.

## Next.js 16 Migration Notes

Key breaking changes from Next.js 15 ŌåÆ 16:

| Change | Impact |
|---|---|
| **`use cache` directive** replaces `unstable_cache` | New explicit caching API ŌĆö mark data functions with `use cache` + `cacheTag` |
| **Implicit caching removed** ŌĆö everything is dynamic by default | Old `fetch` caching patterns deprecated; use `use cache` explicitly |
| **PPR (Partial Prerendering) stable** ŌĆö `cacheComponents: true` | Static shell + streaming dynamic content; no longer experimental |
| **Router scroll optimization** enabled by default | Previously scroll was reset on navigation; now preserved by default |
| Flat config default in `@next/eslint-plugin-next` | ESLint config format change |
| Deprecated `.turbo` config object removed | Use `turbopack` key in `next.config.ts` instead |
| `publicRuntimeConfig` / `serverRuntimeConfig` removed | Use environment variables directly |
| **`next lint` removed** | Use Biome (`npx biome check`) or ESLint (`npx eslint .`) directly |
| **`middleware.ts` deprecated ŌåÆ `proxy.ts`** | `middleware` export renamed to `proxy`; must be `async`; `matcher` moved to `next.config.ts` or exported from `proxy.ts` |

**Upgrade:** `npx @next/codemod@canary upgrade latest` (Next.js 16.2+) ŌĆö automated upgrade CLI that handles codemods, config migrations, and breaking changes in one command. Alternatively `npx next upgrade` (interactive, Next.js 16.1+) or `npm install next@latest`.
